# Alert Component

Alert components provide contextual feedback messages for user actions with built-in icons and semantic colors.

## Available Classes

### Container

- `.alert-wrapper` - Wrapper container with responsive padding (1rem on mobile, 2.5rem horizontal on 40em+)

### Base Alert

- `.alert` - Base alert component with flexbox layout, borders, and icon placeholder

### Alert Types

- `.alert-success` - Success alert with green border and checkmark icon
- `.alert-info` - Info alert with blue border and "!" icon
- `.alert-warning` - Warning alert with yellow/orange border and "!" icon
- `.alert-danger` - Danger/error alert with red border and "✕" icon

## Features

### Built-in Icons

Each alert type includes an automatically generated icon using CSS pseudo-elements:

- **Success**: Checkmark (✓)
- **Info**: Exclamation mark (!) in a circle
- **Warning**: Exclamation mark (!) in a circle
- **Danger**: Cross mark (✕) in a circle

### Responsive Design

The alert wrapper automatically adjusts padding:
- Mobile: 1rem padding on all sides
- Desktop (40em+): 2.5rem horizontal padding

### Semantic Colors

Uses CSS custom properties for consistent theming:
- `--color-success` - Green for success
- `--color-info` - Blue for information
- `--color-warning` - Yellow/orange for warnings
- `--color-danger` - Red for errors

## Dark Mode Support

Alerts automatically adapt to dark mode when used within `.body-dark .form-transparent` contexts, adjusting borders and text colors for better visibility.

## Styling Details

- **Min-width**: 30em
- **Max-width**: 100%
- **Border radius**: 2px (configurable via `--alert-border-radius`)
- **Font size**: 0.8rem
- **Icon size**: 1.5rem
- **Left padding**: 60px (to accommodate icon)

## Best Practices

1. **Use semantic types**: Choose the alert type that matches the message context
   - Success: Confirmations, successful operations
   - Info: Helpful information, tips
   - Warning: Cautions, important notices
   - Danger: Errors, critical issues

2. **Keep messages concise**: Alert text should be brief and actionable

3. **Use wrapper for multiple alerts**: When displaying multiple alerts, wrap them in `.alert-wrapper` for consistent spacing

4. **Position wisely**: Place alerts near the relevant content or at the top of forms

5. **Consider dismissibility**: For non-critical alerts, consider adding a close button

## Accessibility

- Alerts use semantic colors with sufficient contrast
- Icons provide visual reinforcement of message type
- Text is readable at 0.8rem font size
- Works with screen readers (consider adding `role="alert"` for dynamic alerts)
